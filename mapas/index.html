Concepto 01
¿Qué es un Map?

Un Map es una estructura de datos que almacena pares clave → valor. A diferencia de los objetos literales, las claves pueden ser de cualquier tipo (string, número, objeto, función…) y el orden de inserción se preserva. 

// Map: colección de pares clave-valor con claves de cualquier tipo
const mapa = new Map();

// Las claves pueden ser strings, números, objetos, etc.
const claveObjeto = { id: 1 };
mapa.set('nombre',  'Ana');      // clave string
mapa.set(42,         'respuesta'); // clave número
mapa.set(claveObjeto, 'datos');    // clave objeto
Concepto 02
Creación e inicialización

Un Map puede crearse vacío o inicializarse con un iterable de pares [clave, valor], como un array de arrays de dos elementos. 

// Inicialización desde un array de pares [clave, valor]
const precios = new Map([
  ['manzana', 1.5],
  ['naranja', 2.0],
  ['uva',     3.5]
]);

// También desde Object.entries() para convertir objetos a Map
const obj       = { a: 1, b: 2, c: 3 };
const desdeObj  = new Map(Object.entries(obj));
console.log(desdeObj.get('b')); // → 2
Concepto 03
Métodos principales

Los métodos esenciales de Map permiten agregar, leer, verificar y eliminar entradas de manera eficiente. 

const m = new Map();

// .set(clave, valor) → agrega o actualiza una entrada, retorna el Map
m.set('usuario', 'Carlos');

// .get(clave) → devuelve el valor asociado o undefined si no existe
m.get('usuario');       // → 'Carlos'

// .has(clave) → comprueba si existe la clave (boolean)
m.has('usuario');       // → true
m.has('email');         // → false

// .delete(clave) → elimina la entrada (retorna true si existía)
m.delete('usuario');

// .size → propiedad (no método) con el número de entradas
console.log(m.size);    // → 0

// .clear() → elimina TODAS las entradas del Map
m.clear();

Concepto 04
Iteración sobre un Map

Map es iterable: podemos recorrerlo con for...of o sus métodos .keys(), .values() y .entries(). 

const capitales = new Map([
  ['Colombia', 'Bogotá'],
  ['Perú',     'Lima'],
  ['Chile',    'Santiago']
]);

// for...of con destructuring → accede a [clave, valor] directamente
for (const [país, capital] of capitales) {
  console.log(`${país} → ${capital}`);
}

// .keys()   → iterador de claves
// .values() → iterador de valores
// .entries()→ iterador de [clave, valor] (equivale al for...of anterior)
[...capitales.keys()]    // ['Colombia', 'Perú', 'Chile']
[...capitales.values()]  // ['Bogotá', 'Lima', 'Santiago']

// .forEach() → callback (valor, clave, mapa)
capitales.forEach((capital, país) => {
  console.log(país, capital);
});

Concepto 05
WeakMap

Un WeakMap solo acepta objetos como claves. Las referencias son "débiles": si el objeto clave ya no tiene otras referencias, el garbage collector puede liberarlo automáticamente. Útil para datos privados asociados a objetos sin crear fugas de memoria. 

// WeakMap: solo claves objeto, no iterable, sin .size
const metadatos = new WeakMap();

let nodo = { id: 5 }; // objeto usado como clave
metadatos.set(nodo, { visitas: 42, activo: true });

console.log(metadatos.get(nodo)); // → { visitas: 42, activo: true }

// Si nodo se descarta, el GC puede limpiar la entrada automáticamente
nodo = null; // la entrada puede ser recogida por el garbage collector

// WeakMap NO tiene: .size, .keys(), .values(), .forEach()
Concepto 06
Casos de uso prácticos

Map brilla cuando se necesita un diccionario con claves dinámicas, para memoización, contar frecuencias, cachés de funciones o asociar datos extra a objetos DOM. 

// 1. FRECUENCIA DE PALABRAS — contar ocurrencias
function contarPalabras(texto) {
  const frecuencia = new Map();
  for (const palabra of texto.split(' ')) {
    frecuencia.set(palabra, (frecuencia.get(palabra) ?? 0) + 1);
  }
  return frecuencia;
}

// 2. MEMOIZACIÓN — caché de resultados costosos
function memoize(fn) {
  const cache = new Map();
  return (n) => {
    if (cache.has(n)) return cache.get(n); // hit
    const res = fn(n);
    cache.set(n, res);                     // guarda en caché
    return res;
  };
}

const fibonacci = memoize(n => n <= 1 ? n : fibonacci(n-1) + fibonacci(n-2));
console.log(fibonacci(40)); // rápido gracias al caché
