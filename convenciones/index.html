01 — Nomenclatura

camelCase
Para variables y funciones. La primera palabra en minúscula, las siguientes con mayúscula inicial. Es el estilo más común en JS.
variables
javascript 
// ✅ Correcto — camelCase
let userName = "Ana García";
const totalPrice = 199.99;

function getUserAge() { ... }
const calculateTax = () => { ... };

// ❌ Incorrecto
let user_name = "Ana"; // snake_case (no es JS)
let Username = "Ana"; // PascalCase (reservado para clases)
PascalCase
Para clases y constructores. Cada palabra empieza con mayúscula. Le indica a otros devs que deben usar new.
clases
javascript 
// ✅ PascalCase para clases
class UserProfile {
  constructor(name, email) {
    this.name  = name;
    this.email = email;
  }
}

// Instanciar con new
const user = new UserProfile(
  "Ana", "ana@mail.com"
);
UPPER_SNAKE_CASE
Para constantes globales e inmutables. Las mayúsculas advierten que el valor NUNCA debe cambiar.
constantes
javascript 
// ✅ Constantes globales en UPPER_SNAKE
const MAX_RETRIES    = 3;
const API_BASE_URL   = "https://api.example.com";
const DEFAULT_TIMEOUT = 5000; // ms
const COLORS = {
  PRIMARY: "#00e5ff",
  DANGER:  "#ff6b6b"
};
Nombres descriptivos
El nombre debe revelar la intención. Evitar abreviaciones crípticas — el código se lee más de lo que se escribe.
legibilidad
javascript 
// ❌ Nombres crípticos
let d = new Date();
let n = "Juan";
function p(x, y) { ... }

// ✅ Nombres descriptivos
let currentDate = new Date();
let customerName = "Juan";
function calculateDiscount(price, rate) { ... }

02 — Variables

const › let › var
Usa const por defecto. Solo usa let si necesitas reasignar. Evita var — tiene scope confuso y permite redeclaración.
scope
javascript 
// ✅ const — valor no reasignable
const PI = 3.14159;
const user = { name: "Ana" }; // obj mutable

// ✅ let — cuando hay reasignación
let counter = 0;
counter++;

// ❌ var — evitar siempre
var oldStyle = "confuso";
var oldStyle = "permite redeclarar"; // 😬
Una declaración por línea
Declarar una variable por línea mejora la legibilidad, facilita el debugging y hace más claros los diffs en git.
formato
javascript 
// ❌ Múltiples en una línea
let a = 1, b = 2, c = 3;

// ✅ Una por línea — más legible
const firstName = "Ana";
const lastName  = "García";
let   age       = 28;

// ✅ Destructuring cuando aplica
const { name, email } = userData;

03 — Funciones

Arrow Functions
Sintaxis concisa para funciones. No tienen su propio this, ideal para callbacks y métodos de array.
ES6+
javascript 
// Función tradicional
function add(a, b) { return a + b; }

// Arrow function equivalente
const add = (a, b) => a + b;

// Con callbacks — mucho más limpio
const doubled = [1,2,3]
  .map(n => n * 2)      // [2, 4, 6]
  .filter(n => n > 2); // [4, 6]
Responsabilidad única
Cada función debe hacer UNA sola cosa y hacerla bien. Si el nombre tiene "y" o "or", probablemente necesita dividirse.
clean code
javascript 
// ❌ Hace demasiado
function validateAndSaveAndSendEmail(data) { ... }

// ✅ Una responsabilidad c/u
function validateUserData(data) { ... }
function saveToDatabase(data) { ... }
function sendWelcomeEmail(email) { ... }

// Composición clara
async function registerUser(data) {
  validateUserData(data);
  await saveToDatabase(data);
  sendWelcomeEmail(data.email);
}
Parámetros por defecto
Evitar condiciones internas para validar parámetros. Usar valores por defecto directamente en la firma de la función.
ES6+
javascript 
// ❌ Validación interna verbose
function greet(name) {
  const n = name || "Visitante";
  return `Hola, ${n}`;
}

// ✅ Parámetro por defecto
function greet(name = "Visitante") {
  return `Hola, ${name}`;
}

greet();           // "Hola, Visitante"
greet("Ana");      // "Hola, Ana"
Template Literals
Usar backticks para interpolación de cadenas. Más legible que concatenación con +, soporta multilínea.
strings
javascript 
const user = { name: "Ana", age: 28 };

// ❌ Concatenación vieja
const msg1 = "Hola, " + user.name
             + ". Tienes " + user.age + " años.";

// ✅ Template literal
const msg2 = `Hola, ${user.name}.
Tienes ${user.age} años.`;
