01

Sintaxis y Creación

Una expresión regular es un patrón de búsqueda que describe un conjunto de cadenas. En JS se puede crear con literal /patrón/flags o con el constructor new RegExp().

JavaScript
// ── Forma literal (preferida para patrones fijos) ──
const regex1 = /hola/i;          // busca "hola" sin importar mayúsculas

// ── Forma con constructor (útil para patrones dinámicos) ──
const patron = "hola";
const regex2 = new RegExp(patron, "i"); // mismo efecto, patrón como string

// ── Estructura: /  patrón  /  flags ──
//               ↑ delimitadores, no van en el patrón

02

Flags (Modificadores)

Los flags modifican el comportamiento de la búsqueda. Se colocan después del segundo /.

JavaScript
const texto = "Hola hola HOLA";

// g → global: no para en la primera coincidencia
console.log(texto.match(/hola/g));     // ["hola"]

// i → insensible a mayúsculas
console.log(texto.match(/hola/gi));    // ["Hola", "hola", "HOLA"]

// m → multilinea: ^ y $ afectan cada línea
const ml = "línea1\nlínea2";
console.log(ml.match(/^línea/gm));   // ["línea", "línea"]

// s → dotAll: el punto "." también captura saltos de línea
console.log(/a.b/s.test("a\nb"));      // true

// d → índices: agrega indices de inicio/fin al resultado (ES2022)
const m = /cat/d.exec("concatenar");
console.log(m.indices[0]);          // [3, 6]
FlagNombreDescripción
gglobalBusca todas las coincidencias
iignoreCaseNo distingue mayúsculas/minúsculas
mmultiline^ y $ aplican a cada línea
sdotAllEl . captura \n
uunicodeSoporte completo Unicode
vunicodeSetsUnicode con conjuntos (ES2024)
dhasIndicesÍndices de cada grupo (ES2022)
03

Métodos de String y RegExp

JS ofrece métodos tanto en RegExp como en String para ejecutar una regex sobre texto.

JavaScript
const str = "precio: $99, descuento: $15";

// test() → devuelve true/false (más rápido para validar)
/\$\d+/.test(str);                        // true

// exec() → devuelve el primer match con metadatos
/\$(\d+)/.exec(str);                     // ["$99", "99", index:8, ...]

// match() → igual que exec() sin flag g; con g retorna array simple
str.match(/\$\d+/g);                     // ["$99", "$15"]

// matchAll() → iterador con TODOS los matches + grupos (requiere flag g)
[...str.matchAll(/\$(\d+)/g)].forEach(m =>
  console.log(m[1])                      // "99", "15"
);

// search() → índice del primer match, -1 si no hay
str.search(/\$\d+/);                     // 8

// replace() / replaceAll() → sustituir coincidencias
str.replace(/\$(\d+)/g, "€$1");        // "precio: €99, descuento: €15"

// split() → dividir cadena usando regex como separador
"uno1dos2tres".split(/\d/);            // ["uno", "dos", "tres"]

04

Caracteres Especiales

Los metacaracteres representan clases o posiciones genéricas dentro del patrón.

. Cualquier carácter (excepto \n sin /s)
\d Dígito [0-9]
\D No dígito
\w Palabra [a-zA-Z0-9_]
\W No palabra
\s Espacio en blanco
\S No espacio
^ Inicio de cadena / línea
$ Fin de cadena / línea
\b Límite de palabra
* 0 o más veces
+ 1 o más veces
? 0 o 1 vez (opcional)
{n,m}Entre n y m veces
[abc]Conjunto: a, b ó c
[^abc]Negación del conjunto
a|bAlternativa: a ó b
(?:…)Grupo no capturador
05

Grupos de Captura

Los grupos (…) capturan sub-partes del match accesibles por índice o por nombre.

JavaScript
const fecha = "2024-07-15";

// ── Grupos numerados: $1 $2 $3 ──
const rxFecha = /(\d{4})-(\d{2})-(\d{2})/;
const [, año, mes, dia] = rxFecha.exec(fecha);
console.log(`${dia}/${mes}/${año}`);          // "15/07/2024"

// ── Grupos nombrados: (?<nombre>…) → accedidos por .groups ──
const rxNamed = /(?<año>\d{4})-(?<mes>\d{2})-(?<dia>\d{2})/;
const { groups } = rxNamed.exec(fecha);
console.log(groups.mes);                      // "07"

// ── Reemplazar usando grupos nombrados ──
fecha.replace(rxNamed, "$<dia>/$<mes>/$<año>"); // "15/07/2024"

// ── Grupo no capturador (?:…): agrupa sin crear captura ──
/(?:https?):\/\/([\w.]+)/.exec("https://example.com")[1];
// → "example.com" (sin capturar "https")
06

Lookahead y Lookbehind

Las aserciones de posición verifican lo que hay antes o después sin consumir caracteres (zero-width).

JavaScript
// ── Lookahead positivo (?=…): lo que sigue debe coincidir ──
"100€ 200$".match(/\d+(?=€)/g);        // ["100"]

// ── Lookahead negativo (?!…): lo que sigue NO debe coincidir ──
"100€ 200$".match(/\d+(?!€)/g);        // ["10", "200"]  ← parcial

// ── Lookbehind positivo (?<=…): lo que precede debe coincidir ──
"precio: $99 costo: €45".match(/(?<=\$)\d+/g); // ["99"]

// ── Lookbehind negativo (?<!…): lo que precede NO debe coincidir ──
"abc123def456".match(/(?<![a-z])\d+/g); // ["456"]

// Uso práctico: validar contraseña con lookaheads múltiples
const validarPass = /^(?=.*[A-Z])(?=.*\d)(?=.*[!@#$%]).{8,}$/;
// Requiere: ≥1 mayúscula, ≥1 dígito, ≥1 símbolo, mínimo 8 chars
console.log(validarPass.test("Segur0!pass"));  // true
console.log(validarPass.test("debil"));        // false
07

Casos de Uso Reales

Patrones comunes listos para usar: validación de email, URL, teléfono, tarjeta de crédito y más.

JavaScript
// ── Email ──
const rxEmail = /^[\w.+-]+@[\w-]+\.[a-zA-Z]{2,}$/;
rxEmail.test("user@example.com");         // true

// ── URL (http / https) ──
const rxUrl = /^https?:\/\/[\w-]+(\.[\w-]+)+(\/\S*)?$/;
rxUrl.test("https://example.com/path?q=1"); // true

// ── Teléfono Colombia (+57 seguido de 10 dígitos) ──
const rxTel = /^(\+57)?[3][0-9]{9}$/;
rxTel.test("3001234567");                 // true

// ── Tarjeta de crédito (16 dígitos, grupos de 4) ──
const rxCard = /^(\d{4}[\s-]?){4}$/;
rxCard.test("1234 5678 9012 3456");          // true

// ── Extraer hashtags ──
"Aprende #JavaScript y #Regex hoy".match(/#\w+/g);
// → ["#JavaScript", "#Regex"]

// ── Slug URL amigable ──
"¡Hola Mundo! Expresiones Regulares"
  .normalize("NFD").replace(/[\u0300-\u036f]/g, "")
  .toLowerCase().replace(/[^a-z0-9]+/g, "-");
// → "hola-mundo-expresiones-regulares"
