Conceptos Base

Fundamentos de números en JavaScript: tipos, comportamientos especiales y representaciones

01 · Integer

Precisión Entera

Los enteros en JS son precisos hasta 15 dígitos. Más allá de ese límite, los valores pueden ser inexactos.

// Precisión entera: exacta hasta 15 dígitos let exacto = 999999999999999; // ✅ 15 dígitos let inexacto = 9999999999999999; // ⚠️ 16 dígitos console.log(exacto); // 999999999999999 console.log(inexacto); // 10000000000000000
02 · Float

Precisión Flotante

JS usa punto flotante de doble precisión (IEEE 754). Algunos decimales no pueden representarse exactamente en binario.

// Aritmética flotante: resultado inesperado let a = 0.1; let b = 0.2; console.log(a + b); // 0.30000000000000004 // Solución: usar toFixed para redondear console.log((a + b).toFixed(1)); // "0.3"
03 · Coerción

Números + Cadenas

El operador + con cadenas concatena en lugar de sumar. El orden de los operandos importa.

// + con strings: concatenación let r1 = 10 + 20; // 30 (número) let r2 = 10 + "20"; // "1020" (cadena) let r3 = "10" + 20; // "1020" (cadena) let r4 = 10 + 20 + "30"; // "3030" let r5 = "10" + 20 + 30; // "102030"
04 · Strings

Cadenas Numéricas

JS intenta convertir strings a número en operaciones aritméticas (excepto +). Si no puede, devuelve NaN.

// Operaciones con cadenas numéricas let s = "100"; console.log(s * 2); // 200 ✅ convierte console.log(s - 10); // 90 ✅ convierte console.log(s / 4); // 25 ✅ convierte console.log(s + 5); // "1005" ⚠️ concatena // Conversión explícita console.log(Number(s) + 5); // 105
05 · NaN

NaN — No es un Número

NaN (Not a Number) es el resultado de operaciones matemáticas inválidas. Es de tipo number, pero no es igual a sí mismo.

// NaN: resultado de operación inválida let n1 = 100 / "texto"; // NaN let n2 = parseInt("hola"); // NaN // NaN es "contagioso" console.log(n1 + 5); // NaN // typeof NaN === "number" (!) console.log(typeof n1); // "number" // Detectar NaN correctamente console.log(isNaN(n1)); // true console.log(Number.isNaN(n1)); // true
06 · Infinity

Infinito

JS representa valores más allá del rango del tipo Number como Infinity (positivo) o -Infinity (negativo).

// División entre cero → Infinity console.log(1 / 0); // Infinity console.log(-1 / 0); // -Infinity // Desbordamiento numérico let big = Number.MAX_VALUE * 2; console.log(big); // Infinity // Verificar infinito console.log(isFinite(100)); // true console.log(isFinite(big)); // false
07 · Hex

Hexadecimal

JS admite literales en base 16 (prefijo 0x), base 8 (prefijo 0o) y base 2 (prefijo 0b). Útil para colores, permisos, etc.

// Diferentes bases numéricas let hex = 0xFF; // Hexadecimal → 255 let oct = 0o377; // Octal → 255 let bin = 0b11111111; // Binario → 255 console.log(hex, oct, bin); // 255 255 255 // Convertir a hexadecimal console.log((255).toString(16)); // "ff" console.log((255).toString(2)); // "11111111"
08 · Object

Números como Objetos

Con new Number() se crea un objeto Number (no recomendado). Complica comparaciones y ralentiza el código.

// Primitivo vs Objeto Number let primitivo = 42; let objeto = new Number(42); // typeof diferente console.log(typeof primitivo); // "number" console.log(typeof objeto); // "object" // == convierte, === distingue tipo console.log(primitivo == objeto); // true console.log(primitivo === objeto); // false ⚠️

Métodos Básicos

Disponibles en cualquier instancia numérica — se llaman sobre el número directamente

toString()

toString(base)

Convierte un número a cadena de texto. Acepta una base opcional (2–36) para cambiar el sistema numérico.

let n = 255; // Sin argumento → base 10 n.toString(); // "255" // Con base 16 → hexadecimal n.toString(16); // "ff" // Con base 2 → binario n.toString(2); // "11111111" // Con base 8 → octal n.toString(8); // "377"
toExponential()

toExponential(dígitos)

Devuelve el número en notación exponencial (científica) como cadena. Indica cuántos decimales mostrar.

let n = 123456.789; // Notación exponencial automática n.toExponential(); // "1.23456789e+5" // Con 2 decimales n.toExponential(2); // "1.23e+5" // Útil para números muy grandes o pequeños (0.0000045).toExponential(2); // "4.50e-6"
toFixed()

toFixed(decimales)

Formatea el número con un número fijo de decimales y lo devuelve como cadena. Muy usado para monedas y precios.

let precio = 9.8; // 2 decimales (moneda) precio.toFixed(2); // "9.80" let pi = 3.14159265; pi.toFixed(0); // "3" pi.toFixed(4); // "3.1416" (redondea) // Corrige el problema flotante (0.1 + 0.2).toFixed(1); // "0.3" ✅
toPrecision()

toPrecision(dígitos)

Devuelve el número con una cantidad específica de dígitos significativos en total (no solo decimales).

let n = 123.456; // 4 dígitos significativos n.toPrecision(4); // "123.5" // 6 dígitos significativos n.toPrecision(6); // "123.456" // 2 dígitos → notación exponencial n.toPrecision(2); // "1.2e+2"
valueOf()

valueOf()

Devuelve el valor primitivo de un objeto Number. JS lo llama internamente; raramente se usa de forma explícita.

let objetoNum = new Number(42); // Recupera el primitivo del objeto objetoNum.valueOf(); // 42 (number) // JS lo llama automáticamente en operaciones objetoNum + 8; // 50 // Primitivos también tienen valueOf (99).valueOf(); // 99

Métodos Estáticos

Se llaman sobre Number directamente, no sobre una instancia

Number.isFinite()

Number.isFinite()

Comprueba si un valor es un número finito real. Más estricto que el global isFinite() ya que NO convierte el tipo.

Number.isFinite(42); // true Number.isFinite(Infinity); // false Number.isFinite(-Infinity);// false Number.isFinite(NaN); // false // Diferencia: isFinite() global convierte isFinite("42"); // true ⚠️ convierte Number.isFinite("42"); // false ✅ estricto
Number.isInteger()

Number.isInteger()

Verifica si un valor es un número entero (sin parte decimal). No realiza conversión de tipo.

Number.isInteger(10); // true Number.isInteger(10.0); // true (= 10) Number.isInteger(10.5); // false Number.isInteger("10"); // false (string) Number.isInteger(NaN); // false Number.isInteger(Infinity);// false
Number.isNaN()

Number.isNaN()

Comprueba si el valor es exactamente NaN. Más confiable que isNaN() global porque NO convierte el argumento.

Number.isNaN(NaN); // true Number.isNaN(100/"abc");// true Number.isNaN(42); // false // isNaN global convierte: ¡peligroso! isNaN("hola"); // true ⚠️ string→NaN Number.isNaN("hola"); // false ✅ no convierte
Number.isSafeInteger()

Number.isSafeInteger()

Verifica si el número es un entero seguro, es decir, representable exactamente en IEEE 754 (entre ±2⁵³−1).

Number.isSafeInteger(42); // true Number.isSafeInteger(Number.MAX_SAFE_INTEGER); // true // 2^53 NO es seguro Number.isSafeInteger(Math.pow(2,53)); // false Number.isSafeInteger(10.5); // false (decimal)
Number.parseInt()

Number.parseInt()

Convierte una cadena en entero. Lee hasta el primer carácter no numérico. Acepta una base (radix) opcional.

Number.parseInt("42"); // 42 Number.parseInt("42.9"); // 42 (trunca) Number.parseInt("42px"); // 42 (ignora px) Number.parseInt("abc"); // NaN // Con base (hexadecimal) Number.parseInt("ff", 16); // 255
Number.parseFloat()

Number.parseFloat()

Convierte una cadena en número de punto flotante. Preserva la parte decimal, a diferencia de parseInt.

Number.parseFloat("3.14"); // 3.14 Number.parseFloat("3.14abc"); // 3.14 (ignora) Number.parseFloat("10"); // 10 Number.parseFloat("abc"); // NaN // vs parseInt Number.parseInt("3.99"); // 3 Number.parseFloat("3.99"); // 3.99
